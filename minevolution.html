Minevolution History

It all started back sometime in 2014. A CurseForge user known as TheVoid wanted to test the limits of the game. He took the efficiency enchantment. Out of this came a minigame idea. A game about mining faster and fas... Wait this is prisons! Anyways, I assume the idea evolved as the map was built, as all ideas should probably do. It was named Minevolution and released in Minecraft version 1.8.

At some point in the process the map is suggested to LogDotZip. He starts playing it. By the end of the series, Log has an unreleased version of the map. However because the map was built in a version of the game that was very buggy, Log ended up quitting until the bugs could be fixed. This however, never happened.
Sometime later a new beta version of the map was released. It was a complete revamp, and not many people, myself included, enjoyed it as much. After this TheVoid dropped off the face of the Earth with a blog post.

Enter me. In 2018 youtube recommends me a video by LogDotZip from four years previous. Of course I check out the map. As I have a coding passion, of which I didn't fully realize, I tried to fix the bugs. After some basic testing, I realized the issues were with Minecraft 1.8 itself. As Minecraft 1.9, the VERY next update, was the update that re-did command blocks, I had my work cut out.

This era of my work was never public. However, I spent so much time on it. When I updated the map from 1.8 to 1.9 all command blocks were now facing down. Seeing as in 1.8 command blocks needed redstone, but in 1.9 they send a signal themselves, I had to destroy 99% of all command blocks. I re-coded some of them, and spent most of my time on the infinite storage room. However, this version was laggy and nigh on unusable.

This era of my work had a single public release. However, due to being unstable, it has been yeeted. While here I started a full map re-design. As 1.12 added concrete, this was the new best block to use for the main block. However, this ERA was short lived because of an update...

This era of my work has not a single stable release. While I made many attempts, none worked. In this era the /loot command was added, allowing me to do good things. I also provided a complete map rebuild. With all the new blocks, it just worked. However, villagers were not cool.

This era of my work has not yet finished. The current development version is in 1.16. After two years of no updates, and little work, a lot has changed. The entire map has been rebuilt, again. Remember that I only destroyed 99% of all command blocks in the 1.9 ERA? Well there is still ONE remaining command block from version 1.8.
Because villagers are so stupid to work with, This adds a new custom system. I have needed to learn a lot about datapack to make it work, and have worked on a lot of extra datapacks on the side of this.
